# 맥북에서 DB (Prisma) 구조 동기화 가이드

Windows(또는 다른 기기)에서 작업 후 Git으로 코드를 맥북(Mac)으로 가져왔을 때, 소스 코드는 넘어오지만 **데이터베이스의 테이블 구조(스키마)**는 자동으로 동기화되지 않습니다. 

Prisma를 사용 중이므로 아래의 단계에 따라 맥북의 환경 변수를 설정하고 DB 구조를 동기화해야 합니다.

---

## 1. 환경 변수(`.env`) 셋업 

Git은 보통 보안 상의 이유로 `.env` 파일을 추적하지 않으므로, 프로젝트를 처음 내려받은 맥북에는 `.env` 파일이 없을 것입니다.

1. 프로젝트 루트 경로(`d:\AWS\onedeal` 등에 해당하는 Mac 폴더)에 `.env` 파일을 생성합니다.
2. 기존에 작업하던 PC(Windows)의 `.env` 파일에 있는 `DATABASE_URL` 값을 복사해서 맥북의 `.env` 파일에 똑같이 붙여넣습니다.

```env
# .env 파일 예시
DATABASE_URL="postgresql://유저명:비밀번호@호스트:포트/데이터베이스명"
```

> [!IMPORTANT]
> - **로컬 DB 사용 시:** 맥북에도 PostgreSQL(또는 사용 중인 DB)을 설치하고, 맥북 로컬 DB의 접속 정보를 `.env`에 적어야 합니다.
> - **외부 클라우드 DB 사용 시:**(예: AWS RDS, Supabase 등) 기존 Windows PC와 완전히 동일한 `DATABASE_URL`을 사용하시면 됩니다.

---

## 2. 필수 패키지 설치

코드를 새로 Pull 받았다면, 새로운 패키지가 추가되었을 수 있으므로 패키지를 설치합니다. 터미널을 열고 다음 명령어를 실행하세요.

```bash
npm install
```

---

## 3. Prisma 스키마(DB 구조) 동기화

DB 구조를 동기화하는 명령어입니다. 터미널에서 아래의 **A 방법** 혹은 **B 방법** 중 하나를 실행하세요. (현재 프로젝트에 `prisma/migrations` 폴더가 있으므로 **B 방법**을 추천합니다.)

### A 방법: DB 구조를 강제로 동기화하기 (db push)
마이그레이션 기록을 무시하고, 단순히 현재 `schema.prisma` 파일의 내용을 DB에 적용하는 명령어입니다.

```bash
npx prisma db push
```

### B 방법: 마이그레이션 적용하기 (migrate dev) [추천]
`prisma/migrations` 폴더에 생성된 마이그레이션 기록을 바탕으로 DB를 안전하게 동기화하고 적용하는 방법입니다.

```bash
npx prisma migrate dev
```

> [!TIP]
> 위 명령어들을 실행하면 내부적으로 `npx prisma generate` 명령어도 함께 실행되어, 코드베이스(TypeScript)에서 Prisma를 사용할 수 있게 클라이언트 코드가 자동 생성됩니다.  
> 만약 모델 구조를 바꾸지 않고 클라이언트만 재생성해야 한다면 `npx prisma generate`만 실행하면 됩니다.

---

## 4. 로컬 서버 실행

모든 설정이 끝났다면 개발 서버를 실행하여 정상적으로 DB와 연결되고 에러가 없는지 확인합니다.

```bash
npm run dev
```

이제 브라우저에서 로컬호스트(`http://localhost:3000` 등)에 접속하여 기능이 정상적으로 동작한다면 새로운 맥북에서의 DB 셋업이 완료된 것입니다!
